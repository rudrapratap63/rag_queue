# RAG Queue System

A Retrieval-Augmented Generation (RAG) application built with asynchronous background processing. This project uses FastAPI for the API layer, RQ (Redis Queue) with Valkey as the message broker for background tasks, Qdrant for vector storage, and Groq's LLM via the OpenAI SDK for generation.

## 🚀 Architecture

- **API (`server.py`)**: A FastAPI web server that receives user queries and enqueues them as jobs.
- **Message Broker (`valkey`)**: Acts as a drop-in replacement for Redis to manage the job queue.
- **Worker (`queues/worker.py`)**: An RQ background worker that processes the chat queries, performs semantic search against a Qdrant vector database, and generates responses using the `llama-3.3-70b-versatile` model via Groq.

## 🛠️ Prerequisites

- [Docker](https://www.docker.com/) and Docker Compose
- A [Groq](https://groq.com/) API Key
- An existing Qdrant instance running on your host machine (exposed at `http://localhost:6333` / `http://host.docker.internal:6333`).

## ⚙️ Setup and Installation

1. **Clone the repository and locate the project folder.**

2. **Configure Environment Variables:**
   Create a `.env` file in the root of the `rag_queue` directory:
   ```env
   GROQ_API_KEY=your_groq_api_key_here
   # Optional: Override QDRANT_URL if your vector db is hosted elsewhere
   # QDRANT_URL=http://host.docker.internal:6333
   ```

3. **Start the Docker Containers:**
   Build and start the services using Docker Compose:
   ```bash
   docker-compose up -d --build
   ```

   This will spin up three containers:
   - `valkey`: The Redis-compatible message broker (Port 6379).
   - `api`: The FastAPI application (Port 8000).
   - `worker`: The background process handling RAG and LLM calls.

## 📡 API Endpoints

Once the application is running, the API is available at `http://localhost:8000`.

### 1. Health Status
```http
GET /
```
Returns a simple status message indicating the server is running.

### 2. Submit a Chat Query
```http
POST /chat?query=your_question_here
```
**Response:**
```json
{
  "status": "queued",
  "JOB_ID": "1234-abcd-5678-efgh"
}
```

### 3. Retrieve Job Status / Result
```http
GET /job-status?job_id=1234-abcd-5678-efgh
```
**Response:**
```json
{
  "result": "The answer generated by the LLM based on the PDF context..."
}
```

## 🧠 Details on Local Qdrant
This project assumes you already have a `learning_rag` collection populated in your Qdrant database. By default inside the Docker network, it communicates with Qdrant via `http://host.docker.internal:6333`. Ensure your local Qdrant server is running and accessible.
